# Релиз: обращения и смарт-процесс Bitrix24

## Реализовано

- Обязательный телефон при добавлении клиентского портала.
- Отдельный админский чат на каждое новое обращение.
- Название чата: `Название компании, обращение №N`.
- Кнопки Bitrix24 `Обращение закрыто` и `Удалить чат` без дополнительных подтверждений.
- Настройка смарт-процесса из карточки админского портала:
  - выбор смарт-процесса;
  - выбор воронки;
  - сопоставление стадии открытого обращения;
  - сопоставление стадии завершения;
  - выбор ответственного сотрудника.
- Поиск CRM-компании в админском Bitrix24 по телефону клиентского портала.
- Создание элемента смарт-процесса при открытии обращения.
- Дублирование сообщений клиента, ответов техподдержки и системных уведомлений в таймлайн.
- Перевод CRM-элемента на выбранную завершающую стадию при закрытии обращения.
- Фоновый повтор неуспешной CRM-синхронизации без блокировки чата.
- Автоматическая Flyway-миграция `V7__crm_smart_process_integration.sql`.

## Перед первым production-деплоем

1. Сделать резервную копию PostgreSQL.
2. Проверить, что webhook админского портала имеет права `crm`, `user`, `imbot`.
3. После деплоя открыть карточку админского портала и настроить смарт-процесс.
4. У существующих клиентских порталов заполнить телефон клиента.
